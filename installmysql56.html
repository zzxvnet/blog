Centos7安装mysql5.6教程 ​

检查环境 ​

安装好Centos 7.9之后下面来安装Mysql 5.6 ​

首先检查一下系统版本 ​

cat /etc/redhat-release
CentOS Linux release 7.9.2009 (Core)
uname -a
3.10.0-1160.el7.x86_64

查询是否安装mysql rpm -qa|grep -i mysql ​

mysql rpm -qa|grep -i mysql

INFO

mysql-community-libs-5.6.51-2.el7.x86_64

mysql-community-server-5.6.51-2.el7.x86_64

mysql-community-release-el7-5.noarch

mysql-community-common-5.6.51-2.el7.x86_64

Ⅰ 、卸载方法一 卸载前关闭mysql服务 rpm -ev --nodeps mysql-community-release-el7-5.noarch

rpm -ev --nodeps mysql-community-common-5.6.51-2.el7.x86_64

rpm -ev --nodeps mysql-community-client-5.6.51-2.el7.x86_64

rpm -ev --nodeps mysql-community-libs-5.6.51-2.el7.x86_64

rpm -ev --nodeps mysql-community-server-5.6.51-2.el7.x86_64

执行完命令之后再次执行 rpm-qa|grep -i mysql 会发现已经卸载完成。

Ⅱ、卸载方法二 ​

查询mysql所有路径 find / -name mysql ​

[root@localhost ~]# find / -name mysql

/etc/logrotate.d/mysql
/var/lib/mysql
/var/lib/mysql/mysql
/usr/bin/mysql
/usr/lib64/mysql
/usr/share/mysql

然后 rm -rf 上面找到的每个文件夹(*此命令有风险请仔细检查后在使用回车确认) 如：

rm -rf /var/lib/mysql

后再去找出 etc/my.cnf删除它

wget http://repo.mysql.com/mysql-community-release-el7-5.noarch.rpm

关于防火墙的一些操作 ​

关于iptables 防火墙的操作下面总结一下

# 查看防火墙状态
service iptables status  
#停止防火墙
service iptables stop  
#启动防火墙
service iptables start  
#重启防火墙
service iptables restart
 
#永久关闭防火墙
chkconfig iptables off  
#永久关闭后重启
chkconfig iptables on

开启3306端口 ​

vim /etc/sysconfig/iptables
#加入如下代码
-A INPUT -m state --state NEW -m tcp -p tcp --dport 3306 -j ACCEPT

保存退出后后service iptables restart

service iptables restart

关于firewall 防火墙的操作下面总结一下 ​

#查看firewall服务状态
systemctl status firewalld
#查看firewall的状态
firewall-cmd --state

#开启、重启、关闭、firewalld.service服务
#开启
service firewalld start
#重启
service firewalld restart
#关闭
service firewalld stop
#查看防火墙规则
firewall-cmd --list-all 
查询、开放、关闭端口

#查询端口是否开放
firewall-cmd --query-port=3306/tcp
#开放3306端口
firewall-cmd --permanent --add-port=3306/tcp
#移除端口
firewall-cmd --permanent --remove-port=3306/tcp
 
#重启防火墙(修改配置后要重启防火墙)
firewall-cmd --reload

TIP

#参数解释 1、firwall-cmd：是Linux提供的操作firewall的一个工具；

2、–permanent：表示设置为持久；

3、–add-port：标识添加的端口；

创建数据库 ​

1、首先，使用 MySQL 命令行客户端连接到 MySQL 服务器。例如，以下命令将使用 root 用户连接到本地 MySQL 服务器： ​

mysql -u root -p

2、您将被提示输入 root 用户的密码。输入密码并按回车键。 ​

3、然后，您可以使用 CREATE DATABASE 命令来创建一个新的数据库。例如，以下命令将创建一个名为 mydatabase 的新数据库： ​

CREATE DATABASE mydatabase;

4.如果创建成功，MySQL 将输出一条消息，表示该数据库已成功创建。 ​

5.您可以使用 USE 命令来选择要使用的数据库。例如，以下命令将选择名为 mydatabase 的数据库： ​

USE mydatabase;

6.如果您希望在创建数据库时指定字符集和排序规则，可以使用以下命令： ​

CREATE DATABASE mydatabase CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

在这个例子中，我们指定了 utf8mb4 字符集和 utf8mb4_unicode_ci 排序规则。 ​

7.创建数据库后，您可以创建表并向其中插入数据。要了解如何创建表和插入数据，请参考 MySQL 的官方文档或其他在线资源。 ​

注意：为了安全起见，您应该不要使用 root 用户来创建和管理数据库。相反，您应该创建一个有限的特权用户，并使用该用户来管理数据库。 ​

创建新用户 ​

创建数据库的过程中是不需要设置密码的。如果您需要为新创建的 zzxvhub 数据库设置密码，可以创建一个新用户并为其授予访问该数据库的权限，然后使用该用户来连接和管理数据库。 ​

以下是创建一个新用户并为其授予访问 zzxvhub 数据库的权限的示例命令： ​

mysql
CREATE USER 'newuser'@'localhost' IDENTIFIED BY 'password';
GRANT ALL PRIVILEGES ON zzxvhub.* TO 'newuser'@'localhost';
FLUSH PRIVILEGES;

这将创建一个名为 newuser 的新用户，并为其授予访问 zzxvhub 数据库的所有权限。请将 password 替换为您要设置的实际密码。最后，使用 FLUSH PRIVILEGES 命令刷新权限表以确保更改生效。 ​

现在，您可以使用以下命令以新用户身份连接到 zzxvhub 数据库： ​

mysql
mysql -u newuser -p zzxvhub

输入新用户的密码并按回车键即可连接到 zzxvhub 数据库。 ​

授予访问数据库zzxvhub的权限 ​

首先通过root用户登录MySQL,检查用户zzxvhub的权限 ​

然后输入如下命令查询用户权限 ​

根据您提供的输出，SHOW GRANTS FOR 'zzxvhub'@'localhost' 命令的结果只显示了 zzxvhub 用户在所有数据库中的权限为 USAGE，这意味着该用户没有被授予访问任何数据库或表的权限。如果该用户被授予了访问特定数据库或表的权限，其权限应该显示在输出中。 ​

退出Mysql ​

1.在 MySQL 命令行客户端中，输入 exit 命令并按回车键，或者使用快捷键 Ctrl + D 退出客户端。 ​

2.如果您希望在输入 exit 命令后立即退出客户端，可以使用 -e 选项。例如，以下命令将使用 root 用户身份连接到 MySQL 服务器，创建一个新的数据库 mydatabase，然后立即退出客户端： ​

mysql -u root -p -e "CREATE DATABASE mydatabase;"

在这个例子中，我们使用 -e 选项来指定要在客户端中执行的 SQL 命令。 ​

无论您使用哪种方法，退出 MySQL 命令行客户端后，您将返回到系统的终端提示符下。 ​
